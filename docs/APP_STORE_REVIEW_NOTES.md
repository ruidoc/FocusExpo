## 🎯 核心功能说明

### 1. 应用屏蔽功能（核心特性）

**技术实现**:
- 使用 Apple 官方的 **Screen Time API (Family Controls Framework)**
- 基于 `DeviceActivityMonitor` 实现定时自动屏蔽
- 使用 `ShieldConfigurationExtension` 自定义屏蔽页面

**用户流程**:
1. 用户创建专注计划（选择要屏蔽的应用和专注时长）
2. 授予屏幕使用时间权限（必需）
3. 计划开始时，系统自动屏蔽指定应用
4. 专注期间，被屏蔽的应用无法打开
5. 计划结束后，应用自动解除屏蔽

**为什么需要 Screen Time 权限**:
- **唯一的合规方案**: 这是 Apple 官方提供的唯一合规的应用屏蔽方案
- **用户主动控制**: 用户需要明确授权才能使用屏蔽功能
- **隐私保护**: 我们不会收集或上传用户的应用使用数据
- **符合指南**: 完全遵循 Apple App Review Guidelines 4.0 (Design)

---

### 2. 专注计划管理

**功能说明**:
- **一次性任务**: 快速开始任意时长的专注（15分钟 - 4小时）
- **周期任务**: 设置每周固定时间的重复专注计划
- **计划提醒**: 通过通知提醒用户即将开始的专注计划

---

### 3. 数据统计与激励

**功能说明**:
- 展示每日/每周/每月的专注时长统计
- 专注成功率计算
- 可选的成就截图分享功能（需要相册权限）

## 🔐 权限使用说明

### 必需权限

#### 1. 屏幕使用时间权限 (Family Controls)
- **用途**: 执行应用屏蔽功能（核心功能）
- **请求时机**: 用户首次启动应用或创建专注计划时
- **Info.plist Key**: `NSFamilyControlsUsageDescription`
- **审核重点**:
  - ✅ 用户完全主动触发
  - ✅ 不会收集应用使用数据
  - ✅ 不会上传数据到服务器
  - ✅ 仅用于用户自己创建的专注计划

#### 2. 通知权限
- **用途**:
  - 提醒用户即将开始的专注计划
  - 通知专注完成
- **请求时机**: 首次启动应用时
- **可选性**: 用户可以拒绝，不影响核心功能

### 可选权限

#### 3. 相册权限 (Photo Library)
- **用途**: 保存专注成就截图
- **请求时机**: 用户点击"分享成果"按钮时
- **Info.plist Key**:
  - `NSPhotoLibraryAddUsageDescription`
  - `NSPhotoLibraryUsageDescription`

## ⚠️ 常见审核问题预防

### Q1: 为什么需要Screen Time权限？
**A**:
- Screen Time API 是 Apple 官方提供的唯一合规的应用屏蔽方案
- 我们的核心功能就是帮助用户屏蔽分心应用
- 该权限需要用户明确授权，我们无法绕过
- 我们不会滥用该权限收集数据

### Q2: 是否会收集用户的应用使用数据？
**A**:
- **不会**。我们不会读取或上传用户的Screen Time数据
- 我们只使用该权限来执行用户主动创建的屏蔽任务
- 所有数据处理都在本地完成
- 隐私清单文件中已明确声明

### Q3: 应用的独特价值是什么？
**A**:
- **强制力**: 不是提醒，而是真正物理屏蔽应用
- **自动化**: 按计划自动执行，无需手动操作
- **合规性**: 使用官方API，不破解系统
- **用户控制**: 完全由用户自主设置和管理

### Q4: 订阅如何实现？
**A**:
- 使用 Superwall 第三方付费墙服务
- 底层基于 Apple In-App Purchase (StoreKit)
- 完全符合 Apple 的订阅指南
- 提供"恢复购买"和"管理订阅"入口
