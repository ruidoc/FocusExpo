## 文档分类规范

所有文档必须归类到以下目录之一：

| 目录 | 分类 | 说明 | 示例文档 |
|------|------|------|---------|
| `product/` | 产品文档 | **首选阅读**。产品需求、功能规划、PRD、版本规划等 | PRD、功能规格、路线图 |
| `develop/` | 技术文档 | 开发相关的技术实现、架构设计、API 集成、原生模块等 | 原生桥接、支付集成、权限配置 |
| `growth/` | 增长文档 | 用户增长、A/B 实验、数据分析、运营策略等 | 增长计划、实验方案、数据指标 |
| `design/` | 设计文档 | UI/UX 设计规范、交互设计、视觉标准等 | 设计系统、组件规范、主题定义 |
| `publish/` | 发布文档 | App 构建、审核、上架、版本管理等 | 构建指南、审核清单、版本记录 |
| `onboarding/` | 引导文档 | 新用户引导、功能介绍、使用教程等 | 引导流程、功能说明、FAQ |

> **AI 阅读顺序**：了解产品请先读 `product/`，再根据具体问题查阅其他分类。

## AI 操作规范

### 阅读文档时

1. **先读入口**：首先阅读本文件 `docs/index.md` 了解文档结构
2. **按需定位**：根据问题类型定位到对应分类目录
3. **关联阅读**：相关文档可能跨分类，注意关联查找

### 生成文档时

1. **确定分类**：根据文档内容确定归属目录
2. **命名规范**：使用 kebab-case 命名，如 `feature-name.md`
3. **更新入口**：在本文件的「当前文档目录」中添加新文档条目
4. **添加元信息**：文档开头添加标题和简短描述

### 文档模板

新文档应遵循以下结构：

```markdown
# 文档标题

> 一句话描述文档内容

---

## 背景/目的
说明为什么需要这个文档

## 主要内容
文档核心内容

## 相关文档
- [相关文档1](./path/to/doc.md)
- [相关文档2](./path/to/doc.md)
```

### 分类判断指南

| 如果文档涉及... | 归类到 |
|----------------|--------|
| 产品定义、功能规划、PRD、版本路线图、竞品分析 | `product/` |
| 代码实现、API、技术架构、原生模块、第三方库集成 | `develop/` |
| 用户获取、留存、实验、数据指标、运营策略 | `growth/` |
| UI 组件、交互流程、视觉规范、设计资源 | `design/` |
| 构建配置、审核材料、版本发布、应用商店 | `publish/` |
| 用户教程、功能引导、新手流程、帮助文档 | `onboarding/` |
