## 核心理念

1. **价值先行** - Welcome 页直接传递核心价值："专注，不再需要意志力"
2. **渐进式引导** - 从理解系统级锁定 → 体验效果 → 认知翻转到自动化
3. **强制登录** - Step 4 无跳过按钮，必须登录后才能进入首页
4. **清空路由栈** - 完成 onboarding 后使用 `replace`，确保首页是起点

---

## 流程设计（5步）

```
Welcome → Step 0 → Step 1-2 → Step 3 → Step 4 → 登录 → 创建计划 → 首页
                   (场景选择) (权限设置) (快速体验) (价值引导)
```

## 各步骤详细设计

### Welcome 页面

**目标：** 直接传递核心价值 + 差异化

**内容：**
- **主标题（渐变色）：** 专注，不再需要意志力
  - 使用紫色渐变 `['#7A5AF8', '#9B7BFA', '#B794F6']`
  - 字体：32px, font-weight 700
  
- **副标题（两行）：**
  - 定时自动屏蔽分心应用
  - 系统级强制执行

**CTA：** 开始使用 + 右箭头图标

**辅助文案：** 免费 · 2分钟快速体验

**Skip：** 无

**设计原则：**
- 主标题传递价值（不需要意志力），而非功能描述
- 副标题说明如何实现（自动化 + 强制力）
- 简洁、克制、直接

---

### Step 0: GoalSelect（场景选择）

**目标：** 收集用户画像，用于后续个性化文案

**内容：**
- **主标题：** 你最常在什么场景分心？
- **副标题：** 选择一个，为你定制专注方案

**选项（单选）：**
1. **短视频上瘾** - 刷到停不下来
   - 图标：logo-tiktok, 红色
2. **沉迷游戏** - 难以控制时间
   - 图标：game-controller, 青色
3. **学习分心** - 玩手机影响效率
   - 图标：book, 蓝色
4. **其他** - 我只是想专注
   - 图标：ellipsis-horizontal, 橙色

**交互：** 选中后自动进入下一步（300ms 延迟）

**Skip：** 无

**数据传递：** `problem` 参数会传递给后续所有步骤，用于个性化文案

---

### Step 1-2: PermissionSetup（权限设置）

**目标：** 权限授权 + 选择应用（**必须完成**），渐进式引导

**标题：**
- **主标题（个性化）：** 准备屏蔽{场景}应用
  - 短视频 → "准备屏蔽短视频应用"
  - 游戏 → "准备屏蔽游戏应用"
  - 学习 → "准备屏蔽分心应用"
  - 其他 → "准备屏蔽干扰应用"
- **副标题：** 设置权限并选择应用，只需两步

---

#### Step 1: 授权权限

**卡片样式：**
- 左侧圆形标记（36×36px）：数字 1 或绿色对勾
- 标题：授权屏幕时间权限
- 描述：让系统能够强制锁定应用
- 完成后显示：已完成（绿色文字）

**隐私说明（渐进式）：**
- **显示条件：** Step 1 未完成时显示
- **内容：**
  - 为什么需要这个权限？
  - ✓ 实现系统级屏蔽 - 只有获得权限，才能强制锁定应用
  - ✓ 我们看不到你的数据 - 所有屏蔽逻辑在手机本地执行
  - ✓ 你的隐私是安全的 - 无法查看聊天内容、浏览记录
- **隐藏时机：** Step 1 完成后自动隐藏

---

#### Step 2: 选择应用

**卡片样式：**
- 左侧圆形标记（36×36px）：数字 2 或绿色对勾
- 标题（个性化）：选择{场景}应用
  - 短视频 → "选择短视频应用"
  - 游戏 → "选择游戏应用"
  - 学习 → "选择分心应用"
  - 其他 → "选择要屏蔽的应用"
- 描述：
  - Step 1 未完成：请先完成第一步
  - Step 1 完成后：选择 3-5 个最常{使用场景}的

**推荐应用提示（个性化）：**
- **显示条件：** Step 1 完成 && Step 2 未完成
- **样式：** 小字，紧贴步骤下方（不是卡片）
- **内容：**
  - 短视频 → "推荐：抖音、小红书、B站等"
  - 游戏 → "推荐：王者荣耀、原神、和平精英等"
  - 学习 → "推荐：短视频、游戏、社交应用"
  - 其他 → "推荐：短视频、游戏、社交等"

---

**底部按钮：**
- **文案：** "下一步" / "继续体验"（两步完成后）
- **禁用条件：** 两步未全部完成
- **提示：** 两步完成后显示"完成后，将进入 2 分钟快速体验"

**Skip：** 无

**设计亮点：**
1. 渐进式显示：一次只突出一个重点
2. 场景连接：呼应 Step 0 的选择
3. 个性化推荐：降低选择成本
4. 视觉层级：完成的步骤弱化，未完成的步骤突出

---

### Step 3: QuickExperience（快速体验 - Aha Moment）

**目标：** 2分钟专注体验，让用户首次验证系统级锁定的效果

**分两个阶段：**

---

#### 阶段 A：Ready（准备就绪）

**标题：**
- **主标题：** 一切准备就绪
- **副标题（个性化）：** 点击后，{场景}应用将立即被锁定
  - 短视频 → "点击后，短视频应用将立即被锁定"
  - 游戏 → "点击后，游戏应用将立即被锁定"
  - 学习 → "点击后，干扰应用将立即被锁定"
  - 其他 → "点击后，选中的应用将立即被锁定"

**应用展示卡片：**
- 标题：即将锁定
- 内容：应用图标网格（最多显示 9 个）
- 底部：锁定时长：2 分钟

**CTA：** 立即锁定

**设计要点：**
- 建立仪式感："一切准备就绪"
- 明确预期："点击后...将立即..."
- 果断行动："立即锁定"

---

#### 阶段 B：Active（锁定成功）

**庆祝区（顶部）：**
- 绿色对勾图标（48px，带动画）
- **主标题：** 🎉 恭喜，锁定成功！
- **副标题（个性化）：** 你已成功锁定{场景}应用
  - 短视频 → "你已成功锁定短视频应用"
  - 游戏 → "你已成功锁定游戏应用"
  - 学习 → "你已成功锁定干扰应用"
  - 其他 → "你已成功锁定选中的应用"

**倒计时区：**
- 绿色圆环 + 剩余时间（1:45）

**应用图标区：**
- 显示被锁定的应用（前 6 个）

**验证邀请卡片：**
- 标题：💡 想确认锁定效果？
- 内容：
  ```
  现在可以：
  1. 退出这个页面
  2. 尝试打开{具体应用名}
  3. 看看是否能够打开
  
  提示：锁定期间无法打开
  ```
- 具体应用名（个性化）：
  - 短视频 → "抖音或小红书"
  - 游戏 → "王者荣耀或原神"
  - 学习 → "短视频或游戏应用"

**CTA：** 我已确认（ghost 样式）

**Skip：** 无

**设计亮点：**
1. **庆祝 Aha Moment**：顶部醒目庆祝成功
2. **邀请验证**：而非要求验证，降低门槛
3. **明确步骤**：1-2-3 具体操作，降低验证成本
4. **具体化**：用真实应用名，而非"这些应用"

---

### Step 4: ValueGuide（价值引导 / 认知翻转）

**目标：** 从"手动开启"翻转到"定时自动执行"的价值认知

#### 页面结构

**1. 顶部（承认成功）**
- 绿色对勾图标（48px）
- 文案：你已经体验到了系统级锁定的效果

**2. 问题区**
```
但每次都要手动开启

每次问自己"要不要开始专注？"
这个决定本身就在消耗意志力
```

**3. 解决方案标题**
```
更好的方式：让它每天自动发生
```

**4. 定时计划卡片**
- 标题：定时计划
- 内容：3 个时间段举例（根据场景个性化第一条）
  ```
  每天 9:00 → 自动锁定短视频
  每天 14:00 → 自动锁定游戏
  每天 21:00 → 自动锁定所有干扰
  ```
- 说明：到点自动执行，无需手动开启

**5. 功能点**
```
[📅]        [⏰]        [⚡]
设置一次     每天自动     不用纠结
```

**6. 底部操作区**
- **主按钮：** 登录后创建计划
- **小字说明：** 设置一次，每天生效
- **隐私协议勾选**
- **无跳过按钮**

#### 文案设计原则

1. **承认爽感**：正向强化 Aha Moment
2. **指出痛点**：手动开启消耗意志力
3. **给出方案**：定时计划 = 自动版的刚刚体验
4. **具体场景**：用真实时间段和应用名
5. **强化核心价值**：不需要做决定 = 不消耗意志力

#### 登录后流程

**1. 登录成功：** `router.replace('/plans/add?from=onboarding')`

**2. 创建计划页（特殊模式）：**
- 顶部：无返回按钮（`headerLeft: () => null`）
- 手势：禁用返回（`gestureEnabled: false`）
- 底部按钮：
  - 主按钮：[创建计划]
  - 次按钮：[稍后创建]（文字链接，仅 onboarding 显示）

**3. 退出方式：**
- 创建成功：`router.replace('/(tabs)')` - 清空路由栈
- 稍后创建：`router.replace('/(tabs)')` - 清空路由栈

**4. 进入首页：**
- 路由栈只有首页
- 无法返回到 onboarding

**Skip：** 无（必须登录）

## 优化总结

### 关键改进点

| 优化项 | 优化前 | 优化后 | 改进效果 |
|--------|--------|--------|----------|
| **Welcome 价值传递** | 功能描述 | "专注，不再需要意志力" | 直接传递核心价值 |
| **Step 1-2 引导** | 一次性展示 | 渐进式显示 | 降低认知负担 |
| **Step 3 庆祝** | "限制生效中" | "🎉 恭喜，锁定成功！" | 强化 Aha Moment |
| **Step 4 逻辑** | 文案冗长 | 问题→方案，简洁直接 | 认知翻转更清晰 |
| **跳过选项** | 有跳过按钮 | 必须登录 | 提高转化率 |
| **路由栈管理** | 保留历史 | replace 清空 | 首页是起点 |

### 完整用户旅程

```
Welcome: "专注，不再需要意志力"
  ↓ 建立预期
Step 0: 场景选择（个性化起点）
  ↓ 收集画像
Step 1-2: 渐进式权限设置
  ↓ 建立认知：系统级强制屏蔽
Step 3: "🎉 恭喜，锁定成功！"
  ↓ 验证：强制屏蔽真的有效（Aha Moment）
Step 4: "但每次都要手动开启..." → "让它每天自动发生"
  ↓ 认知翻转：手动 → 自动
登录 → 创建计划（无返回） → 首页（清空路由栈）
```
