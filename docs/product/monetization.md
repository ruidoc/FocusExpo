# FocusVow 商业化策略（中国区）

> 版本：v1.3
> 目标市场：中国大陆 iOS App Store
> 当前实现：自研 Paywall + `expo-iap` + Apple Webhook

---

## 1. 核心策略：纯订阅制 (Subscription Only)

基于 FocusVow “持续自律” 的产品属性，我们采用**纯订阅制**。

* **移除买断**：不再提供终身买断选项，优先保证长期现金流与持续迭代动力。
* **多周期组合**：提供周、月、年三种周期，覆盖不同决策成本的用户。
* **低门槛准入**：通过免费试用和首周特惠，降低首次付费决策门槛。

---

## 2. 权益对比 (Feature Gating)

通过限制**数量**和**高级功能**来设置付费点。

| 功能模块 | 免费版 (Free) | 专业版 (Pro) | 设计意图 |
| :--- | :--- | :--- | :--- |
| **屏蔽应用数** | **限制 3 个** | **无限量** | 3 个仅够屏蔽最常用分心应用，重度用户会明显受限。 |
| **专注计划数** | **限制 2 个** | **无限量** | 免费版只能覆盖极少场景，无法细分工作、学习、阅读等。 |
| **专注时长** | 单次上限 45 分钟 | **无限时长** | 超过 45 分钟的深度专注需求更接近付费意愿人群。 |
| **统计报表** | 仅查看**今日**数据 | **全历史趋势** (周/月/年) | 想复盘和看到长期进步的用户必须升级。 |
| **严格模式** | 不支持 | **支持** | 免费版允许放弃/暂停，Pro 才是真正不可轻易中断的契约。 |
| **白名单模式** | 不支持 | **支持** | “仅保留微信和电话，其他全屏蔽”属于高阶需求。 |
| **应用图标** | 默认图标 | **专属图标** | 满足个性化需求。 |

## 3. 定价策略 (Pricing)

采用“高频低价入口 + 长期高性价比”的组合策略。

### 3.1 基础定价方案 (Baseline)

| 套餐类型 | 价格 (CNY) | 优惠/试用 | 促销文案 | 策略意图 |
| :--- | :--- | :--- | :--- | :--- |
| **年度订阅** | **¥78 / 年** | **7 天免费试用（新用户仅一次）** | 每天仅 ¥0.21，用更低成本换来一整年的稳定专注 | **主推选项**，通过免费试用降低顾虑，锁定长期用户。 |
| **月度订阅** | **¥12 / 月** | 无 | 灵活订阅，随时取消 | **中间选项**，给不想长期承诺的用户留退路。 |
| **周订阅** | **¥6 / 周** | **首周 ¥1（新用户仅一次）** | 首周特惠，仅需 1 元立即体验 | **低门槛入口**，吸引价格敏感用户先迈出第一步。 |

### 3.2 当前实现说明

* 当前实际上架配置为：**年包 ¥78 / 年，7 天免费试用；月包 ¥12 / 月，无优惠；周包 ¥6 / 周，首周 ¥1**。
* 年包与周包都属于 Apple 推介促销优惠，**新用户仅可享受一次**。
* 客户端商品列表以 `/product/list` 返回结果为准，价格、套餐排序、优惠信息应与该接口和 `App Store Connect` 保持一致。
* 当前客户端支付页仍有“3 天免费试用”文案残留，必须按实际上架配置统一为“7 天免费试用”或更保守的优惠文案。

## 4. 当前入口策略

当前版本已经移除 Superwall，订阅主链路统一到客户端自研 Paywall。

### 4.1 当前订阅入口

* **我的页会员卡片**：用户点击会员卡片或“升级”按钮后进入 `paywall/index`。
* **会员中心**：未订阅用户点击“立即订阅”后进入 `paywall/index`。
* **恢复购买**：在 `paywall/index` 和 `user/rights` 中都提供恢复购买入口。
* **管理订阅**：在 `user/rights` 中跳转系统订阅管理页。

### 4.2 当前展示策略

* 客户端先读取后端商品列表，再结合 App Store 返回的本地化价格进行展示。
* 当前默认选中 `period` 最大的套餐，实际效果等同于优先推荐年包。
* 当前是固定版 Paywall，不支持远程模板、远程 placement 或远程 A/B 分流。

## 5. 技术实现（当前版本）

### 5.1 客户端链路

客户端当前基于 `expo-iap` 完成订阅主流程：

* 通过 `/product/list?product_class=focus&is_subscription=1` 获取商品配置。
* 使用 `expo-iap` 拉取 App Store 商品信息并展示本地化价格。
* 用户点击购买后调用 `requestPurchase({ type: 'subs' })` 发起订阅。
* 购买成功后，客户端短轮询 `/subscription` 判断订阅是否已入库。
* 支持 `restorePurchases()` 恢复购买。

### 5.2 用户关联方式

* 购买时会将 `user.superwall_uuid` 作为 `appAccountToken` 传给 StoreKit。
* 虽然字段名仍是历史命名，但当前真实用途是 **Apple Webhook 关联用户**。
* 后端收到 Apple 通知后，会优先用 `appAccountToken` 查用户，兜底再用 `original_transaction_id` 查历史订阅。

### 5.3 服务端链路

服务端当前以 Apple Server Notification V2 为订阅事实来源：

* `SUBSCRIBED`：创建或更新订阅记录。
* `DID_RENEW`：处理续费或试用转正。
* `DID_CHANGE_RENEWAL_STATUS`：处理关闭/恢复自动续费。
* `EXPIRED` / `REFUND` / `REVOKE`：处理到期、退款、撤销。
* `/subscription`：返回当前用户的有效订阅。

### 5.4 当前实现边界

* 当前没有 RevenueCat，也没有 Superwall。
* 当前没有远程可配 Paywall、远程实验分流或运营模板能力。
* 当前订阅体验完全依赖客户端页面 + Apple 支付 + 自有后端入库。

## 6. 验证清单

### 当前 IAP 上线前必测项目

* [ ] **商品展示正确**
  - `/product/list` 返回的商品与 `App Store Connect` 商品一致
  - 客户端展示价格与 App Store 本地化价格一致
  - 默认选中的套餐符合当前策略

* [ ] **首次购买成功**
  - 用户可正常拉起 App Store 支付
  - 支付成功后 `/subscription` 能查到有效订阅
  - 会员中心与我的页会员态同步更新

* [ ] **恢复购买**
  - 删除 App、重装或切换账号后可恢复购买
  - 恢复后 `/subscription` 与会员 UI 同步恢复

* [ ] **试用链路**
  - 试用开始后 `trial_end_at` 正确
  - 试用期内不应创建付费记录
  - 试用转正后订阅价格、支付记录正确写入

* [ ] **续费、取消、到期**
  - 自动续费后 `expires_at` 正确延长
  - 关闭自动续费后，当前周期内权益仍然可用
  - 到期后会员权益正确失效

* [ ] **退款链路**
  - 退款后订阅和支付状态正确变更
  - UI 能正确反映退款后的非会员状态

* [ ] **Webhook 幂等性**
  - 重复投递同一 Apple 通知不会重复入库

## 7. 当前未完善点

### 7.1 文案与商品配置未完全对齐

* 当前支付页文案写的是 **3 天免费试用**。
* 但当前实际上架配置已经明确为：**年包 7 天免费试用（新用户一次）、月包无优惠、周包首周 ¥1（新用户一次）**。
* 必须以 `App Store Connect` 和 `/product/list` 实际配置为准统一文案，否则会误导用户或影响审核。

### 7.2 Apple Webhook 仍缺正式验签

* 当前服务端只做 JWS 解码，没有完成 Apple 官方证书链验签。
* 这会影响生产环境安全性和通知可信度。

### 7.3 取消自动续费的有效期判断仍有风险

* 当前后端把关闭自动续费写成 `cancelled`。
* 但 `/subscription` 的有效订阅判定目前只认 `active`。
* 这会导致用户在“已取消自动续费但仍在有效期内”时被错误识别为非会员。

### 7.4 购买成功后的客户端兜底不足

* 当前客户端购买成功后只会短轮询 `/subscription` 3 次。
* 如果 Apple 通知延迟或服务端入库稍慢，用户会看到“已支付但权益没立即生效”。

### 7.5 历史命名尚未收口

* `superwall_uuid` 这个字段名仍在客户端和服务端中使用。
* 当前它的真实用途是 Apple IAP 的 `appAccountToken`。
* 功能上还能工作，但语义已经不清晰，后续建议重命名为更明确的字段。

### 7.6 优惠资格展示仍需谨慎

* 年包和周包的优惠都带有“**新用户仅一次**”前提。
* 但当前客户端展示主要依赖 `/product/list` 和本地商品信息，**未明确区分用户当前是否仍具备该优惠资格**。
* 如果前端无法准确判断资格，文案应避免绝对承诺，或在购买前补充“符合条件的新用户可享”。

---

## 8. 建议的下一步

1. 先统一客户端文案、`product/list` 返回值和 `App Store Connect` 实际配置。
2. 修复“取消自动续费但未到期”的权益判断。
3. 给 Apple Webhook 补正式验签。
4. 为购买成功增加更稳的客户端兜底同步策略。
