# FocusVow 商业化策略（中国区）

> 版本：v1.2
> 目标市场：中国大陆 iOS App Store
> 核心工具：Superwall + IAP

---

## 1. 核心策略：纯订阅制 (Subscription Only)

基于 FocusVow "持续自律" 的产品属性，我们采用**纯订阅制**。

*   **移除买断**：不再提供终身买断选项。这有助于维持长期的现金流，并持续激励团队迭代产品。
*   **多周期组合**：提供周、月、年三种周期，满足不同决策成本的用户。
*   **低门槛准入**：通过"免费试用"和"首周特惠"大幅降低用户的首次体验门槛，促使犹豫用户迈出第一步。

---

## 2. 权益对比 (Feature Gating)

通过限制**数量**和**高级功能**来设置付费点。

| 功能模块 | 免费版 (Free) | 专业版 (Pro) | 设计意图 |
| :--- | :--- | :--- | :--- |
| **屏蔽应用数** | **限制 3 个** | **无限量** | 3个仅够屏蔽最常用的（如抖音、微信、小红书），重度手机用户不够用。 |
| **专注计划数** | **限制 2 个** | **无限量** | 2个仅够设置"工作日"和"周末"，无法细分"早晨阅读"、"午休"等场景。 |
| **专注时长** | 单次上限 45 分钟 | **无限时长** | 番茄钟通常25-45分钟，超过此需求的用户通常是深度学习/工作者，付费意愿强。 |
| **统计报表** | 仅查看**今日**数据 | **全历史趋势** (周/月/年) | 想要复盘和看到长期进步的用户必须付费。 |
| **严格模式** | 不支持 | **支持** | 免费版允许"放弃/暂停"（扣自律币）；Pro版支持"无法取消/无法暂停"，真正的"契约"。 |
| **白名单模式** | 不支持 | **支持** | "只允许微信和电话，其他全屏蔽"，这是高阶需求。 |
| **应用图标** | 默认图标 | **专属图标** | 个性化需求。 |

## 3. 定价策略 (Pricing)

采用 **"高频低价入口 + 长期高性价比"** 的组合策略。

### 3.1 基础定价方案 (Baseline)

| 套餐类型 | 价格 (CNY) | 优惠/试用 | 促销文案 | 策略意图 |
| :--- | :--- | :--- | :--- | :--- |
| **年度订阅** | **¥88 / 年** | **7天免费试用** | 每天仅 ¥0.24，一杯咖啡钱换一年自律 | **主推选项 (Best Value)**。通过免费试用消除顾虑，锁定长期用户。 |
| **月度订阅** | **¥15 / 月** | 无 | 灵活订阅，随时取消 | **中间选项**。给不想长期承诺的用户一个退路，锚定年费的性价比。 |
| **周订阅** | **¥6 / 周** | **首周 ¥1** | **首周特惠**，仅需 1 元立即体验 | **低门槛入口**。利用极低的首周价格吸引价格敏感型用户进入，后续转化为周费或升级年费。 |

### 3.2 价格心理学分析

1.  **年订阅 (¥88)**：
    *   ¥88 / 12 ≈ ¥7.3/月。
    *   对比月订阅 (¥15)，相当于 **5 折**。
    *   加上 **7天免费试用**，是理性用户的首选。

2.  **周订阅 (¥6)**：
    *   ¥6 * 4 = ¥24/月。比月订阅贵，但单次支付额极低。
    *   **首周 ¥1** 是核心杀手锏：用户心理是"花1块钱试用一周Pro功能"，决策成本几乎为零。

## 4. A/B 测试策略 (利用 Superwall)

利用 Superwall 的 Paywall 远程配置能力，测试不同的 Paywall 展示逻辑和套餐组合。

*   **实验组 A (经典试用)**：
    *   展示：**年 (试用) + 月**。
    *   默认选中：年。
    *   目标：测试标准 SaaS 的转化模型。

*   **实验组 B (低价诱导)**：
    *   展示：**年 (试用) + 周 (首周特惠)**。
    *   默认选中：周。
    *   目标：测试"首周 ¥1"对转化率的提升效果。

*   **实验组 C (全选项)**：
    *   展示：**年 + 月 + 周**。
    *   默认选中：年。
    *   目标：给予用户最多选择，观察用户自然偏好。

## 5. Superwall 投放策略 (Placements)

不要在 App 启动时直接弹窗（除非是回流用户），要在用户**产生需求**或**感受到价值**时展示。

### 5.1 场景一：功能触发 (Feature Gating)
*最精准的转化场景*

*   **Trigger**: 用户尝试添加第 4 个屏蔽应用。
*   **Paywall**: "解锁无限应用" 主题。
*   **文案**: "想屏蔽更多？升级 Pro 解锁无限应用数量，彻底告别干扰。"

*   **Trigger**: 用户尝试创建第 3 个计划。
*   **Paywall**: "解锁无限计划" 主题。
*   **文案**: "为不同场景定制专属计划，工作、学习、阅读互不干扰。"

*   **Trigger**: 用户点击"历史统计"或"周报"。
*   **Paywall**: "数据洞察" 主题。
*   **文案**: "见证你的每一次进步，查看完整的专注趋势分析。"

### 5.2 场景二：价值感知峰值 (Aha Moment)
*情绪价值最高的场景*

*   **Trigger**: 用户**首次完成**一次时长 > 25分钟 的专注。
*   **Paywall**: "庆祝/鼓励" 主题。
*   **文案**: "太棒了！你刚刚专注了 25 分钟。坚持下去，升级 Pro 让自律更简单。"
*   **Offer**: 此时可以展示 **周订阅 (首周 ¥1)**，文案："奖励自己，1元体验一周 Pro 会员"。

### 5.3 场景三：Onboarding 结束
*筛选高意向用户*

*   **Trigger**: 新手引导流程结束，进入首页前。
*   **Paywall**: "全面介绍" 主题。
*   **文案**: "开启你的专注契约。现在试用，7天内免费。"
*   **默认选中**: 年度订阅（带试用）。

✅ **修复类型错误**
- 清理了 `order.controller.ts`、`payment.service.ts`、`subscription.service.ts` 中的 `status` 类型不匹配问题
- 编译通过，无类型错误

### 7.2 客户端修复

✅ **修复 identify 时序问题**
- 在 `SuperwallProviderWrapper` 中添加 `onConfigured` 回调
- SDK 初始化完成后自动重新 identify（兜底逻辑）
- 确保无论何时启动，用户都会被正确识别
- 路径：`src/components/providers/SuperwallProvider.tsx`

✅ **现有 identify 调用点**

## 8. 验证清单

### 上线前必测项目

- [ ] **周包首周折扣**：
  - 用户购买周包（首周 ¥1），检查 `subscriptions.period = 0.25`
  - 检查 `subscriptions.price = 100`（1元 * 100分）
  - 检查 `payments.amount = 100`

- [ ] **年包免费试用**：
  - 用户开始试用，检查 `subscriptions.trial_end_at` 正确
  - 检查 `orders.is_trial = 1`
  - 检查**不会创建 `payments` 记录**（试用期不扣款）

- [ ] **试用转正**：
  - 试用期结束后，收到 `RENEWAL` 事件
  - 检查 `subscriptions.trial_end_at` 被清空
  - 检查 `subscriptions.price` 更新为正价
  - 检查**创建了 `payments` 记录**

- [ ] **Superwall identify**：
  - 冷启动时检查日志：`[Superwall] 兜底识别xxx用户: {userId}`
  - 登录后检查日志：`[Superwall] 用户识别: {userId}`
  - 确认 Webhook 中 `originalAppUserId` 不是 `$SuperwallAlias:`

- [ ] **Webhook 幂等性**：
  - 重复发送同一个 `event_id` 的订阅事件
  - 确认返回 `Event already processed`，不会重复入库
