# 「专注契约」创业计划书

---

## 一、技术可行性分析

### 1. 项目名称

**专注契约（FocusVow）** —— 杀死手机瘾，拯救专注力！

### 2. 核心技术与创新点

**产品定位：** 一款不靠意志力、无痛戒断手机瘾的自律专注APP。

**技术实现：**

基于 iOS Family Controls 框架，具备两大核心能力：**系统性强制**——在系统层面锁定用户选定的分心应用，由系统服务执行，用户无法绕过；**定时锁定**——支持设定时间段与重复规则，到点自动执行锁定，无需每次手动启动。

**两种专注模式：**

| 模式                       | 说明                                   | 场景                               |
| -------------------------- | -------------------------------------- | ---------------------------------- |
| **快速专注**（一次性任务） | 选择应用 → 设置时长 → 立即开始         | 临时需要专注、新手体验             |
| **专注计划**（定时任务）   | 设定时间段和重复规则，系统每天自动执行 | 每天固定时段自动屏蔽，无需手动启动 |

**核心创新：**

1. **系统级物理阻断**：不是提醒，是屏蔽期间应用真正无法打开——市面上大多数专注工具只能"提醒"
2. **自动执行，无需意志力**：定时计划设置一次后每天自动生效，将意志力投入转化为一次性配置
3. **自律小组 / 监督小组**：邀请好友组建自律小组，组队自律、互相监督；可查看组内成员专注完成情况，形成同伴压力与正向激励，提升留存与口碑传播
4. **三层状态同步**：JS → Native Bridge → iOS Extension，通过 App Groups 实现跨进程实时通信

### 3. 国内外现状

**核心问题：** 现有专注工具要么只能"提醒"（Forest、番茄 TODO），要么仅支持 Android（不做手机控），要么本地化差、定价高（Opal $99.99/年）。

国内 iOS 市场存在明确空白：**缺乏一款中文原生、基于 Screen Time API 的强制屏蔽工具。**

**趋势：**

- 中国用户日均手机使用 5+ 小时，数字健康成为刚需
- iOS 15 开放 Family Controls API，系统级屏蔽工具的技术条件已成熟
- 用户从"提醒式"向"强制式"工具迁移的意愿增强

### 4. 当前开发状态

- 代码 10,000+ 行，30+ 屏幕，已完成核心功能
- 后端 NestJS 服务已部署生产环境
- TestFlight 内测进行中
- App Store IAP 订阅已集成

### 5. 可靠性与安全性

- **错误监控**：Sentry 实时告警
- **数据存储**：三层架构（MMKV 高频缓存 + AsyncStorage 偏好 + App Groups 跨进程）
- **安全**：JWT 认证 + IAP 服务端校验 + Webhook 签名验证
- **隐私**：仅调用 Apple 官方 API，不采集具体应用使用内容，符合 App Store 隐私规范

### 6. 知识产权

- 软件著作权申请中
- 「专注契约」商标注册申请中
- App Store 审核准备中

## 二、市场可行性分析

### 1. 市场调查

**市场规模：**

- 中国 iOS 用户约 2.5 亿
- 数字健康类应用全球市场预计 2026 年达 $80 亿
- 国内目标市场估算 30-50 亿人民币

**竞品对比：**

| 维度         | 专注契约  | Forest    | Opal      | 番茄 TODO |
| ------------ | --------- | --------- | --------- | --------- |
| 强制屏蔽     | ✅ 系统级 | ❌ 仅提醒 | ✅ 系统级 | ❌        |
| 中文体验     | ✅ 原生   | ✅        | ❌ 英文   | ✅        |
| 自动定时执行 | ✅        | ❌        | ✅        | ❌        |
| 年订阅价     | ¥88       | ¥25       | ≈¥700     | ¥98       |

**结论：** 唯一具备"中文原生 + 系统级屏蔽 + 自动执行 + 合理定价"的产品。

### 2. 目标用户与需求

**目标用户：**

| 人群                | 特征                     | 核心痛点                                             |
| ------------------- | ------------------------ | ---------------------------------------------------- |
| 备考学生（18-25）   | 考研、考公、期末考试     | 知道该学习，总在"要不要开始"上纠结，缺乏启动意志力   |
| 深度工作者（25-35） | 知识工作者、远程办公     | 固定时段需专注，但总忘记手动启动，提醒类工具无效     |
| 自我提升者（20-40） | 有个人成长追求           | 想减少刷手机，感觉被手机控制，亟需强制戒断手段       |

**需求验证：** 中国用户日均手机使用约 6 小时，短视频等场景让不少人刷到停不下来；提醒类工具（Forest、番茄钟）留存有限，用户已从"提醒式"向"强制式"迁移。痛点真实、需求可被验证。

### 3. 商业模式

**盈利方式：** 纯订阅制（周/月/年），通过功能限制（屏蔽应用数、计划数、统计范围、严格模式等）驱动升级。

**定价逻辑：** 年订 ¥88 主推（含 7 天免费试用），月订 ¥15 灵活锚定，周订 ¥6（首周 ¥1）低门槛转化。年费相当于月费 5 折，理性用户首选年订。

**单位经济：** 目标 LTV/CAC > 3；首年依赖自然增长与口碑，获客成本可控；自律小组上线后，邀请裂变可降低边际 CAC。

### 4. 获客策略

**主渠道：** App Store ASO（关键词：专注、屏蔽手机、戒手机瘾、防沉迷）；小红书/抖音/B 站学习区内容合作。

**辅助渠道：** 自律小组邀请机制——用户邀请好友组队监督，被邀请人完成首次专注即达成，邀请人可获得权益，形成自然裂变。

**阶段策略：** 第一年以 ASO + 内容为主，验证转化与留存；自律小组上线后，口碑传播占比提升，目标 K 因子 0.1-0.2。

### 5. 市场预测

**短期（1-2 年）：** 数字健康类应用市场增速维持 10%+；用户从提醒式工具向强制式工具迁移意愿增强，国内 iOS 强制屏蔽类产品供给仍稀缺；手机依赖问题持续存在，刚需不减。

**中期（3-5 年）：** 数字健康全球市场规模预计突破 $80 亿，国内目标市场 30-50 亿；强制式专注工具品类认知度提升，竞品可能增多；若 Android 厂商开放类似 API，市场可进一步扩容。
