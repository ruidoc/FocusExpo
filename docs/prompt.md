# 武侠暗号 - 代码生成提示词系统

## 使用说明

在请求代码生成时，在消息的**最后新加一行**使用对应的"武侠武器"暗号，AI 将自动按照相应的模式进行响应。

## 武器暗号与生成模式

### 🗡️ **长剑** - 纯分析模式

**暗号**: `长剑`
**特点**: 只分析不动手，深入思考实现逻辑

- 分析实现方式和技术方案
- 梳理参数获取方式
- 识别潜在问题和注意事项
- **绝不修改任何代码**
- 输出详细的逻辑分析报告

### ⚔️ **宝剑** - 分析+核心代码模式

**暗号**: `宝剑`
**特点**: 分析后展示关键代码片段

- 先进行完整的实现分析
- 输出核心代码逻辑（伪代码或关键片段）
- 说明代码的关键实现点
- **不直接修改项目文件**
- 提供可参考的代码示例

### 🏹 **神弓** - 精准执行模式

**暗号**: `神弓`
**特点**: 严格按需求执行，不做扩展

- 完全按照用户要求生成代码
- 不添加额外功能或优化
- 不进行代码重构或美化
- 精准命中目标需求
- 最小化修改范围

### 🔨 **重锤** - 完善增强模式

**暗号**: `重锤`
**特点**: 在用户需求基础上完善和扩展

- 实现用户的核心需求
- 补充必要的边界处理
- 添加合理的错误处理
- 优化代码结构和性能
- 增加适当的注释说明

### 🗡️ **匕首** - 快速修复模式

**暗号**: `匕首`
**特点**: 快速定位问题，最小化修改

- 快速识别和定位问题
- 用最少的代码修改解决问题
- 不进行大范围重构
- 保持现有代码风格
- 专注于问题解决

### 🛡️ **盾牌** - 防御优化模式

**暗号**: `盾牌`
**特点**: 注重代码安全性和健壮性

- 添加必要的参数验证
- 增强错误处理机制
- 考虑边界条件和异常情况
- 提升代码的容错能力
- 关注性能和内存安全

### ⚡ **闪电剑** - 极速开发模式

**暗号**: `闪电剑`
**特点**: 快速生成可用代码，后续可迭代

- 快速实现基础功能
- 优先保证代码可运行
- 使用最直接的实现方式
- 减少复杂的设计模式
- 为后续迭代留出空间

### 🌟 **七星剑** - 完美主义模式

**暗号**: `七星剑`
**特点**: 追求代码质量和最佳实践

- 遵循最佳编程实践
- 考虑代码的可维护性
- 添加完善的文档注释
- 实现优雅的设计模式
- 注重代码的可读性和扩展性

## 使用示例

```
帮我分析一下如何实现用户登录功能

#长剑
```

```
实现一个带输入验证的用户注册表单

#重锤
```

```
只需要一个简单的计数器组件，不要添加其他功能

#神弓
```

## 注意事项

1. 暗号必须放在消息的**最后一行**，格式为 `#武器名`
2. 暗号前面必须空一行，与问题内容分开
3. 如果不使用暗号，将采用默认的平衡模式
4. 可以在对话中切换不同的暗号模式
5. 暗号格式：`#长剑`、`#神弓`、`#重锤` 等

## 默认模式

如果没有使用任何暗号，AI 将采用平衡模式：

- 适度分析实现方案
- 生成完整可用的代码
- 包含基础的错误处理
- 添加必要的注释说明
